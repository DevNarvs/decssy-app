# Plan 1: Foundation & Auth — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a Next.js 15 app with Convex + Clerk authentication, the Peach Fuzz design system, a 4-tab adaptive navigation shell, and PWA installability — so a user can sign up, see four empty tab placeholder screens, and sign out.

**Architecture:** App-Router Next.js 15 with TypeScript strict mode. Convex provides the database and reactive query layer; Clerk provides authentication with a JWT bridge into Convex (`auth.config.ts`). User identity syncs from Clerk to a Convex `users` table on first authenticated request. The UI shell uses bottom-tab navigation on mobile (320–767px) and a left sidebar on tablet+ (≥768px); both are placeholders pointing to empty pages that future plans will fill in. PWA installability is added via a web manifest and a minimal service worker. Schema in this plan is intentionally minimal — only `users`, `groups`, `groupMembers` — because Plan 1 ships *auth that works*, not *features*.

**Tech Stack:**
- Next.js 15 (App Router), React 19, TypeScript 5 strict
- Convex (database + reactive queries + serverless functions)
- Clerk (`@clerk/nextjs`) for authentication
- Tailwind CSS 3.4 with custom Peach Fuzz tokens, `tailwindcss-animate` for shadcn primitives
- shadcn/ui (configured but no components imported yet)
- Plus Jakarta Sans via `next/font/google`
- Lucide React for icons
- Vitest + @testing-library/react for component unit tests
- Playwright for the one end-to-end auth smoke test

**Prerequisites (one-time setup before Task 0):**
1. **Node.js 20.x or 22.x LTS** installed (`node -v` to verify)
2. **A Clerk account** at clerk.com (free tier is fine) — create a new application called "Decssy"; note the publishable key and secret key
3. **A Convex account** at convex.dev (free tier is fine) — `npx convex` will sign you in via browser on first run
4. **A Vercel account** at vercel.com — only needed at deploy time (Task 17), not earlier
5. **PowerShell 7+** recommended on Windows (works with `&&`); PowerShell 5.1 works too but command chaining differs

## Task 9: User sync — getCurrentUser query and store mutation

**Files:**
- Create: `convex/users.ts`

- [ ] **Step 1: Create convex/users.ts**

Create `convex/users.ts`:

```ts
import { v } from "convex/values";
import { mutation, query, type QueryCtx, type MutationCtx } from "./_generated/server";
import type { Doc } from "./_generated/dataModel";

/**
 * Resolves the currently signed-in Clerk identity to a Convex user row.
 * Returns null if the user has not been synced yet.
 */
export const getCurrentUser = query({
  args: {},
  handler: async (ctx) => {
    return await getCurrentUserOrNull(ctx);
  },
});

/**
 * Idempotent upsert called from the client on first authenticated render.
 * If the Clerk identity has no Convex user row yet, create one.
 * If it exists, update mutable fields (name, avatar) only.
 */
export const storeUser = mutation({
  args: {
    timezone: v.string(),
  },
  handler: async (ctx, { timezone }) => {
    const identity = await ctx.auth.getUserIdentity();
    if (!identity) {
      throw new Error("Called storeUser without authentication");
    }

    const existing = await ctx.db
      .query("users")
      .withIndex("by_clerkId", (q) => q.eq("clerkId", identity.subject))
      .unique();

    if (existing) {
      // Update mutable fields if they changed in Clerk.
      const patch: Partial<Doc<"users">> = {};
      if (existing.name !== (identity.name ?? "")) patch.name = identity.name ?? "";
      if (existing.email !== (identity.email ?? "")) patch.email = identity.email ?? "";
      if (existing.avatarUrl !== identity.pictureUrl) patch.avatarUrl = identity.pictureUrl;
      if (Object.keys(patch).length > 0) {
        await ctx.db.patch(existing._id, patch);
      }
      return existing._id;
    }

    const userId = await ctx.db.insert("users", {
      clerkId: identity.subject,
      email: identity.email ?? "",
      name: identity.name ?? identity.email ?? "Anonymous",
      avatarUrl: identity.pictureUrl,
      timezone,
    });
    return userId;
  },
});

// ── Internal helpers (not exported as Convex API) ─────────────────────────────

export async function getCurrentUserOrNull(
  ctx: QueryCtx | MutationCtx,
): Promise<Doc<"users"> | null> {
  const identity = await ctx.auth.getUserIdentity();
  if (!identity) return null;
  return await ctx.db
    .query("users")
    .withIndex("by_clerkId", (q) => q.eq("clerkId", identity.subject))
    .unique();
}

export async function requireCurrentUser(
  ctx: QueryCtx | MutationCtx,
): Promise<Doc<"users">> {
  const user = await getCurrentUserOrNull(ctx);
  if (!user) {
    throw new Error("Not signed in or user not yet synced");
  }
  return user;
}
```

## Task 10: useStoreUserEffect hook (client-side first-sync trigger)

**Files:**
- Create: `lib/useStoreUserEffect.ts`

- [ ] **Step 1: Create the hook**

Create `lib/useStoreUserEffect.ts`:

```ts
"use client";

import { useEffect, useState } from "react";
import { useAuth } from "@clerk/nextjs";
import { useConvexAuth, useMutation } from "convex/react";
import { api } from "@/convex/_generated/api";
import type { Id } from "@/convex/_generated/dataModel";

/**
 * Calls storeUser exactly once after Clerk + Convex auth are both ready.
 * Uses the browser's IANA timezone via Intl.DateTimeFormat.
 * Returns the Convex userId once stored, or null while pending.
 */
export function useStoreUserEffect(): { userId: Id<"users"> | null; isLoading: boolean } {
  const { isLoading, isAuthenticated } = useConvexAuth();
  const { isSignedIn } = useAuth();
  const [userId, setUserId] = useState<Id<"users"> | null>(null);
  const storeUser = useMutation(api.users.storeUser);

  useEffect(() => {
    if (!isAuthenticated || !isSignedIn) return;

    let cancelled = false;
    const tz = Intl.DateTimeFormat().resolvedOptions().timeZone || "UTC";

    storeUser({ timezone: tz })
      .then((id) => {
        if (!cancelled) setUserId(id);
      })
      .catch((err) => {
        console.error("storeUser failed", err);
      });

    return () => {
      cancelled = true;
    };
  }, [isAuthenticated, isSignedIn, storeUser]);

  return { userId, isLoading: isLoading || (isAuthenticated && userId === null) };
}
```

```tsx
"use client";

import { useEffect } from "react";

export function ServiceWorkerRegistrar() {
  useEffect(() => {
    if (typeof window === "undefined") return;
    if (!("serviceWorker" in navigator)) return;
    if (process.env.NODE_ENV !== "production") return;

    const register = () => {
      navigator.serviceWorker
        .register("/sw.js", { scope: "/" })
        .catch((err) => console.warn("SW registration failed:", err));
    };

    if (document.readyState === "complete") {
      register();
    } else {
      window.addEventListener("load", register, { once: true });
    }
  }, []);

  return null;
}
```
